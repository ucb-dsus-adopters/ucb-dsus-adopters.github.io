# How to Adopt

The standardized workflow for bringing data science curriculum and infrastructure to your institution.

</div>

Every course on this site follows the **same adoption workflow**. Modules have a lighter
process (download notebooks and go). This guide is the standardized, single source of
truth for the course workflow; each [course page](../courses/index.md) then composes these
steps with its own repositories, hub, license, and any course-specific notes. For modules,
see the [Modules](../modules/index.md) section.

Most instructors are up and running in **a couple of hours** once they have GitHub access
and import the Canvas template. If a JupyterHub is being provisioned for you (for example
through [Cloudbank](https://www.cloudbank.org/) or
[Cal-ICOR](https://www.cal-icor.org/)), that step can take longer depending on
scheduling. We will tell you what to expect when you get in touch.

## What you'll get

When you adopt a course you can expect:

- The Jupyter notebooks for labs, homeworks, and projects
- The solutions to all the notebooks (private)
- Lecture slides and, where available, videos
- Worksheets (and solutions) used in discussion/lab sessions
- A Canvas template customized to use JupyterHub for your institution
- Access to a JupyterHub instance configured to your campus single sign-on
- Access to a grading service that grades student notebooks and returns a CSV of grades

## The workflow at a glance

1. **Choose a delivery mode:** pick how students will run notebooks (campus JupyterHub,
   a partner-provisioned hub via [Cloudbank](https://www.cloudbank.org/) or
   [Cal-ICOR](https://www.cal-icor.org/), or alternatives like Colab/Binder). See
   [Infrastructure & Tools](infrastructure.md).
2. **Get access:** create a GitHub account, complete the course's instructor interest
   form, fork the student materials, and accept the solutions + grading invitations.
3. **Set up Canvas:** download the course's Canvas template, point its links at your
   hub, and import it. See [Canvas Import](canvas-import.md).
4. **Grade with Otter:** use the web-based Otter Service Standalone (or Gradescope /
   local). See [Otter Autograding](autograding.md).
5. **Adapt content:** customize within the bounds of each course's license. See
   [Adapting Content](adapting-content.md) and [Understanding Licenses](licenses.md).
6. **Get support:** see [Support](support.md) when you need help.
