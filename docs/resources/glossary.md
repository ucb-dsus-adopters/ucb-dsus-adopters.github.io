# Glossary

**Autograder / `autograder.zip`**
: The Otter solution bundle for an assignment. You download it from a course's private
solutions repository and feed it to the grader along with student submissions.

**Canvas common cartridge (`.imscc`)**
: A standard package format for importing course content (structure, assignment links,
quizzes) into Canvas or another LMS.

**Cal-ICOR**
: California Interactive Computing Open Resource
([cal-icor.org](https://www.cal-icor.org/)), supported by the California Education Learning
Lab. Cal-ICOR provides cloud-hosted computing infrastructure (including JupyterHubs) for
California public colleges and universities. See also the
[Cal-ICOR modules](../modules/cal-icor.md) page.

**Cloudbank**
: An NSF-funded program ([cloudbank.org](https://www.cloudbank.org/)) that provisions cloud
computing infrastructure, including JupyterHubs, for institutions nationwide.

**Common cartridge JupyterHub rewriter**
: A tool that rewrites the hub URLs and materials-repo references inside a Canvas template
so assignments point at *your* JupyterHub before import.

**JupyterHub**
: A multi-user server that gives each student a Jupyter notebook environment in the
browser, typically behind campus single sign-on.

**LMS (Learning Management System)**
: The platform that hosts your course (e.g., Canvas) where students access assignments and
submit work.

**Otter / otter-grader**
: The autograding framework used to grade Jupyter notebooks. Some courses pin a specific
version (e.g., Data 6 requires 6.1.6).

**Otter Service Standalone**
: The web-based Otter grading service at grader.datahub.berkeley.edu, with no local install.

**Solutions repository (private)**
: The private GitHub repo holding solutions and `autograder.zip` files for a course. Access
is granted after completing the instructor interest form.
